<objective>
Fix the Server Action auth token forwarding gap in both admin pages so that PUT calls to the Fastify API succeed instead of returning 401 Unauthorized.

Purpose: AUTH-06 (admin role management) and TIER-04 (admin tier configuration) are both blocked by the same root cause — Server Actions and GET fetches in admin pages send no Authorization header. <!-- Root cause documented in VERIFICATION.md:
  Both admin pages use Next.js Server Actions to call the Fastify API.
  The Server Actions make fetch() calls with only Content-Type: application/json.
  Fastify's requireRole(['admin']) decorator calls getAuth(request) from @clerk/fastify,
  which requires a valid Clerk JWT in the Authorization: Bearer <token> header.
  Without it the requests return 401 Unauthorized.

  Fix: use auth() from @clerk/nextjs/server to obtain the session token, then
  forward it as Authorization: Bearer ${token} in every fetch call to the API.

  Note: auth() is imported from @clerk/nextjs/server (already used in lib/auth.ts).
  The getToken() method returns the JWT for the current session.
  Pattern: const { getToken } = await auth(); const token = await getToken();
-->
